Cookie+Session+localStroage+SessionStorage

 2018/04/10   Share

说明

cookie

1、服务器通过Set-Cookid头给客户端一串字符串
2、客服端每次访问相同域名的网页时，必须带上这段字符串
3、客户端要在一段时间内保存这个Cookid
4、Cookie 默认在用户关闭页面后就失效，后台代码可以任意设置 Cookie 的过期时间
5、大小大概在 4kb 以内

Session

1、将sessionID（随机数）通过cookie发给客户端
2、客户端访问服务器时，服务器读取sessionID
3、服务器有一块内存（哈希表）保存了所有session
4、通过 SessionID 我们可以得到对应用户的隐私信息，如 id、email
5、这块内存（哈希表）就是服务器上的所有 session

LocalStorage

1、LocalStorage 跟 HTTP 无关
2、HTTP 不会带上 LocalStorage 的值
3、只有相同域名的页面才能互相读取 LocalStorage（没有同源那么严格）
4、每个域名 localStorage 最大存储量为 5Mb 左右（每个浏览器不一样）
5、常用场景：记录有没有提示过用户（没有用的信息，不能记录密码）
6、LocalStorage 永久有效，除非用户清理缓存

SessionStorage（会话存储）

1、2、3、4 同上
SessionStorage 在用户关闭页面（会话结束）后就失效。

各自的区别

Cookie和Session 的区别

一般来说session是基于cookie实现的，sessionID（随机数）通过cookie发给客户端
区别
1、cookie数据存放在客户的浏览器上。session数据放在服务器上，通过 SessionID 我们可以得到对应用户的隐私信息，如 id、email
2、cookie不是很安全，应当使用session。
3、session会在一定时间内保存在服务器上。当访问增多，会比较占用你服务器的性能
4、单个cookie保存的数据不能超过4K，很多浏览器都限制一个站点最多保存20个cookie。
5、session不能区分路径，同一个用户在访问一个网站期间，所有的session在任何一个地方都可以访问到。而cookie中如果设置了路径参数，那么同一个网站中不同路径下的cookie互相是访问不到的。
6、session中保存的是对象，cookie中保存的是字符串。

Cookie和LocalStorage 的区别

1、服务器通过Set-Cookid头给客户端一串字符串。LocalStorage 跟 HTTP 无关，HTTP 不会带上 LocalStorage 的值，LocalStorage 不会随 HTTP 发给 Server
2、Cookie大小大概在 4kb 以内。每个域名 localStorage 最大存储量为 5Mb 左右（每个浏览器不一样）
3、Cookie一般由服务器生成，可设置失效时间，如果在浏览器生成，默认是关闭浏览器之后失效。LocalStorage用于浏览器端缓存数据，除非被清楚，否则永久保存

LocalStorage 和 SessionStorage 的区别

LocalStorage 除非被清楚，否则永久保存。SessionStorage 在用户关闭页面（会话结束）后就失效。
