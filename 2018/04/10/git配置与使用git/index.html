配置git

使用git,一定要运行这五句话，不然会很坑爹。

使用git

初始化和一些git的命令

1、创建目录作为我们的项目目录：mkdir git-demo-1
2、进入目录 cd git-demo-1
3、git init，这句命令会在 git-demo-1 里创建一个 .git 目录
4、运行 git status -sb 可以看到文件前面有 ?? 号

5、使用 git add 将文件添加到「暂存区」。(git add index.html增加一个index.html的文件添加到「暂存区」。git add .是将所有的文件添加到「暂存区」)
6、使用 git commit -m "信息"将你 add 过的内容「正式提交」到本地仓库（.git就是本地仓库），并添加一些注释信息，方便日后查阅
7、如果有新的变动，我们只需要依次执行 git add xxx 和 git commit -m ‘xxx’ 两个命令即可。其实就这一句有用！先 add 再 commit，行了，你学会 git 了。
8、git log 查看变更历史

将本地仓库上传到 GitHub

1、在 GitHub 上新建一个空仓库，直接创建就好了，名称随意，什么都不用改，一般可以跟本地目录名一致，也叫做 git-demo-1，
2、点击创建按钮之后，GitHub 就会把后续的操作全告诉
3、我使用ssh的，所以选择ssh命令行。在上面已经初始化了本地仓库，所以选择第二种方式，git remote add origin git@github.com:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/git-demo-1.git，复制并运行它。复制第二行 git push -u origin master，运行它（第一种是比较多的命令行）
4、刷新当前页面，你的仓库就上传到 GitHub 了！

如何上传更新

1、git add 文件路径
2、git commit -m “信息”
3、git pull （相信我，你一定会忘记这一个命令）
4、git push
下面是例子
cd git-demo-1
touch index2.html
git add index2.html
git commit -m “新建 index2.html”
git pull
git push
然后你去 git-demo-1 的 GitHub 页面，就能看到 index2.html 出现在里面了。

git其他一些有用的命令

git remote add origin git@github.com:xxxxxxx.git将本地仓库与远程仓库关联
git remote set-url origin git@github.com:xxxxx.git 上一步手抖了，可以用这个命令来挽回
git branch 新建分支
git merge 合并分支
git stash 通灵术
git stash pop 反转通灵术
git revert 后悔了
git reset 另一种后悔了
git diff 查看详细变化
